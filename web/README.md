## 📖 AI-Powered CHW eTraining Platform <a name="about-project"></a>

> Empowering Community Health Workers, improving gender equity, and enhancing access to care in Rwanda through an AI-powered, digitalized eTraining system for high quality Universal Health Coverage.

**AI-Powered CHW eTraining Platform** is a comprehensive digital health system designed to revolutionize community health worker training, patient care coordination, and health service delivery in Rwanda. The platform integrates AI-powered diagnostics, automated certification systems, and seamless communication channels to bridge the gap between communities and healthcare providers while promoting gender equity and improving access to quality healthcare services.

Design and Development of eTraining / eLearning Portal and AI-Powered Models for Community Health Workers and eventual Community EMR Integration:
- Deliver a custom eTraining / eLearning portal for Community Health Workers (CHWs).
- Integrate predictive AI tools to enhance early diagnosis, health risk analysis, and georeferencing.
- Support digital supervision, remote communication, and capacity building for CHWs and patients.

### Key Features <a name="key-features"></a>

- **Cloud-Based eTraining / eLearning Portal**: Scalable and secure platform accessible via web and mobile with offline content access and auto-synchronization
- **AI-Powered Diagnostics & Predictive Analytics**: Machine learning models for early diagnosis, health risk analysis, TB default prediction, and maternal complications assessment
- **Comprehensive User Management**: Multi-role system supporting admin, trainer, CHW, and patient accounts with appropriate permissions
- **Interactive eLearning Experience**: Multimedia content support (video, audio, slides) with quiz and assessment engines providing real-time feedback
- **Digital Supervision & Communication**: Remote supervision capabilities with seamless communication channels between CHWs, supervisors, and patients
- **Automated Certificate Generation**: Digital certification system with automatic CHW registration in national databases
- **Georeferencing & Clinical Decision Support**: Location-based health analytics with AI-powered alerts, reminders, and recommended actions
- **Community EMR Integration**: Secure API integration with community Electronic Medical Records for comprehensive health data management

## 📋 Project Phases <a name="project-phases"></a>

### Phase I: Inception and Planning
- **Stakeholder Consultation**: Conduct comprehensive needs assessment through stakeholder consultations, document review, and field visits
- **Inception Report**: Draft detailed methodologies, technologies, and project implementation plan
- **Initial Design**: Create initial mockups for stakeholder review and approval

### Phase II: Prototype Development
- **UX/UI Design**: Develop comprehensive wireframes and interactive platform mockups
- **Clickable Prototype**: Deliver interactive prototype for feedback and usability testing

### Phase III: eTraining / eLearning Portal MVP Development
- **Platform Development**: Design and develop cloud-based, scalable MVP with core functionalities:
  - User account management (admin, trainer, CHW, patient roles)
  - Multimedia content support (video, audio, slides)
  - Quiz and assessment engines with real-time feedback
  - Certificate generation system
  - Offline content access and auto-synchronization
- **MVP Deployment**: Deploy to test environment with comprehensive documentation

### Phase IV: AI Model Design and Data Integration
- **Data Collection**: Collect and pre-process health data from community EMR for model training
- **AI Model Development**: Design and develop AI models for:
  - Diagnostic support and clinical decision making
  - Georeferencing and location-based analytics
  - Predictive analytics (TB default, maternal complications)
  - Clinical decision support (alerts, reminders, recommended actions)
- **EMR Integration**: Ensure secure API integration with community EMR backend
- **Model Validation**: Conduct comprehensive testing, debugging, and model validation

### Phase V: Final Implementation and Training
- **Stakeholder Training**: Train stakeholders' staff and develop technical documentation
- **Documentation**: Create comprehensive user manuals and technical documentation
- **Capacity Building**: Knowledge transfer and training for stakeholders' team, project staff, and selected trainers

## 💻 Getting Started <a name="getting-started"></a>

To get a local copy up and running, follow these steps.

### Prerequisites

In order to run this project you need:

```sh
# Node.js (v18 or higher)
node --version

# Python (v3.8 or higher)
python --version

# PostgreSQL
psql --version

# Git
git --version
```

### Setup

Clone this repository to your desired folder:

```sh
cd my-folder
git clone git@github.com:WelTel-Rwanda/weltel-rwanda-chw-etraining.git
cd weltel-rwanda-chw-etraining
```

### Install

Install this project with:

```sh
# Install backend dependencies
cd backend
pnpm install

# Install Python dependencies for AI services
pip install -r requirements.txt

# Install frontend dependencies
cd ../frontend
pnpm install

# Install mobile app dependencies
cd ../mobile
npm install
```

### Usage

To run the project, execute the following command:

```sh
# Start the backend server
cd backend
pnpm run dev

# Start the frontend development server
cd frontend
pnpm run dev

# Start the mobile app (requires Expo CLI)
cd mobile
npx expo start
```

### Run tests

To run tests, run the following command:

```sh
# Run backend tests
cd backend
npm test

# Run frontend tests
cd frontend
npm test

# Run AI model tests
cd ai-services
python -m pytest tests/
```

### Deployment

You can deploy this project using:

```sh
# Build for production
npm run build

# Deploy to cloud platform (e.g., AWS, Azure)
npm run deploy
```

## ❓ FAQ <a name="faq"></a>

- **How does the AI model training and validation process work?**
  - We collect and pre-process health data from community EMR systems with proper consent from MoH/RBC/Fionet. The AI models are trained on validated datasets and undergo rigorous testing, debugging, and validation before deployment.

- **What are the core functionalities of the eTraining / eLearning platform?**
  - The platform includes user account management for multiple roles, multimedia content support, interactive quiz and assessment engines, automated certificate generation, offline content access, and auto-synchronization capabilities.

- **How is the platform integrated with existing EMR systems?**
  - We ensure secure API integration with community EMR backends, following all regulatory requirements and obtaining necessary consents from health authorities for data access and integration.

- **What types of AI models are being developed?**
  - Our AI models focus on diagnostic support, georeferencing, predictive analytics (including TB default and maternal complications), and clinical decision support with alerts, reminders, and recommended actions.

- **How is capacity building handled for stakeholders?**
  - We provide comprehensive training for stakeholders' staff, develop detailed technical documentation, create user manuals, and conduct knowledge transfer sessions for project staff and selected trainers.
